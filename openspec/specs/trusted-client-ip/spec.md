# trusted-client-ip 规范

## 目的

为云端服务（Cloudflare Worker）在「直连 Cloudflare 源站 vs 经阿里云 CDN 回源」双入口架构下，统一解析**可信的真实客户端 IP**，供所有以 IP 为安全计数键的机制（限流、后续防爬的 per-IP 自适应难度与会话配额）取用。以阿里云 CDN 注入的不可伪造密钥回源头（`X-Origin-Auth`）判定「请求确来自 CDN」，再采信 CDN 写入的真实 IP 头（`Ali-Cdn-Real-Ip`），绝不采信客户端可伪造的 `X-Forwarded-For`；配置缺失时一律 fail-safe 到 `CF-Connecting-IP`。该解析产物仅作限流/防爬计数键（抬高自动化批量调用成本），不用作访问控制/鉴权。

## 需求
### 需求：可信真实客户端 IP 解析

云端服务（Cloudflare Worker）**必须**提供统一的「可信真实客户端 IP」解析逻辑，供所有以 IP 为安全计数键的机制（限流、后续防爬的 per-IP 自适应难度与会话配额）取用。该解析**必须**正确处理双入口架构——直连 Cloudflare 源站（`qsl.herbertgao.me`）与经阿里云 CDN 回源（`qsl.herbert-dev.cn` → 源站），且**绝不**无条件信任客户端可控的请求头。

> **边界**：本解析产物**仅**用作限流/防爬计数键（抬高自动化批量调用成本），**禁止**用作访问控制/鉴权判据；鉴权由 Bearer Key 与请求签名承担。

**信任根（解析正确性的前提）：**
- (R1) `CF-Connecting-IP` 可信，当且仅当请求必经且仅经 Cloudflare 边缘、由 CF 边缘按 TCP 层对端写入并覆盖客户端发送的同名头（当前架构 Cloudflare 之前无其他代理层）。
- (R2) 「请求确来自阿里云 CDN 回源」由**密钥回源头 `X-Origin-Auth`** 判定：阿里云 CDN 以「修改出站请求头」覆盖语义注入固定密钥（覆盖客户端伪造的同名头），worker 持有期望密钥 `CDN_ORIGIN_SECRET` 并以**常量时间**比对。**禁止**用回源 IP 白名单判定——阿里云回源 IP 动态分配、官方不建议固定白名单。
- (R3) 采信的真实用户 IP 取自 CDN 写入的真实 IP 头（推荐 `Ali-Cdn-Real-Ip`，阿里云默认携带、为防 `X-Forwarded-For` 伪造而设）；其「覆写而非透传」须经部署期证伪门实证。

判定规则：

- **当** `CF-Connecting-IP` 缺失或为空 → 可信真实客户端 IP **必须**为 `'unknown'`。
- **当** 期望密钥 `CDN_ORIGIN_SECRET` **未配置/为空**（未启用采信注入头）→ **必须**返回 `CF-Connecting-IP`，**禁止**采信任何注入头。
- **当** 请求的 `X-Origin-Auth` 经常量时间比对**不等于**期望密钥（直连，或来源不可信，或伪造密钥）→ **必须**返回 `CF-Connecting-IP`，**禁止**采信任何注入头（`X-Forwarded-For`、`Ali-Cdn-Real-Ip` 等在此场景均可由客户端伪造）。
- **当** `X-Origin-Auth` 密钥校验**通过**（确来自 CDN）：
  - 若受信真实 IP 头名 `CDN_REAL_IP_HEADER` **未配置** → **必须** fail-safe 返回 `CF-Connecting-IP`（头名待证伪门确认后再配）。
  - 否则取该头值；经运行时校验（单值/合法 IP 字面量/不含逗号）通过 → **必须**返回该值（去空白）；缺失/为空/多值/非法 → **必须**返回 `'unknown'`（惩罚桶，**禁止**回退 `CF-Connecting-IP` 节点桶）。
- 任何路径**绝不**采信 `X-Forwarded-For`（含其首段/任意段——XFF 是 append 语义、首段为客户端可伪造值）。

> 说明：本需求取代「`CF-Connecting-IP` 不可伪造」这一在前置 CDN 架构下不再成立的旧前提；信任信号是 CDN 注入的不可伪造密钥头（客户端伪造被 CDN 覆盖、攻击者猜不到密钥），而非动态、不可稳定白名单的回源 IP。

#### 场景：经 CDN（有效密钥）采信 CDN 写入的真实 IP 头

- **当** 请求携带有效 `X-Origin-Auth`（等于 `CDN_ORIGIN_SECRET`）
- **并且** 已配 `CDN_REAL_IP_HEADER`，该头为合法单值真实用户 IP
- **那么** 可信真实客户端 IP **必须**为该头的值（去空白）
- **并且** 限流/计数键**必须**以该真实用户 IP 区分不同用户，而非归并到 CDN 回源节点 IP（即便同一真实用户经不同 CDN 节点 IP 到达，也**必须**归同一桶）

#### 场景：经 CDN 路径下忽略客户端伪造的 XFF

- **当** 请求携带有效 `X-Origin-Auth` 且携带客户端伪造的 `X-Forwarded-For`
- **那么** 解析**禁止**采用 `X-Forwarded-For` 任何分段
- **并且** 可信真实客户端 IP **必须**取自 CDN 写入的受信真实 IP 头（伪造 XFF 不能制造无限自选限流桶）

#### 场景：无密钥/错误密钥时忽略一切注入头

- **当** 请求无 `X-Origin-Auth`，或其值经常量时间比对不等于 `CDN_ORIGIN_SECRET`（直连或伪造密钥）
- **并且** 请求携带客户端伪造的 `Ali-Cdn-Real-Ip` 与/或 `X-Forwarded-For`
- **那么** 可信真实客户端 IP **必须**为 `CF-Connecting-IP`
- **并且** **禁止**采用任何客户端可注入头（伪造密钥/伪造 IP 头均不能绕过或污染限流计数）

#### 场景：密钥未配置时 fail-safe

- **当** `CDN_ORIGIN_SECRET` 未配置或为空
- **并且** 请求携带 `X-Origin-Auth` / `Ali-Cdn-Real-Ip`
- **那么** 解析**必须**使用 `CF-Connecting-IP`、忽略一切注入头
- **并且** **禁止**因缺密钥配置而采信任何客户端可控头

#### 场景：密钥正确但真实头缺失/多值回退惩罚桶

- **当** `X-Origin-Auth` 密钥校验通过且已配 `CDN_REAL_IP_HEADER`
- **并且** 该头缺失、为空、非合法 IP 字面量、或为多值（含逗号——覆写假设失效信号）
- **那么** 可信真实客户端 IP **必须**为 `'unknown'`
- **并且** **禁止**回退为 `CF-Connecting-IP`（避免坍缩进 CDN 回源节点 IP 桶）

#### 场景：无任何有效 IP 回退 unknown

- **当** `CF-Connecting-IP` 缺失或为空
- **那么** 可信真实客户端 IP **必须**为 `'unknown'`（共享惩罚桶）

### 需求：密钥回源头与受信真实 IP 头的配置与确认

采信 CDN 注入头的合规性依赖「该头被 CDN 覆写而非透传」这一**单点信任根**。**运行时无法区分**「CDN 覆写写入的单值合法头」与「客户端透传进来的同名单值合法头」；故「配置的头确被 CDN 覆写」**必须**由**部署期证伪门**保证（运行时校验仅作纵深防御、只能挡多值/非法这类「透传+追加」的直接信号，**禁止**以运行时校验通过等同该值可信）：

- **密钥配置**：`CDN_ORIGIN_SECRET` 为机密，**必须**经 Secret 注入（`wrangler secret put`），**禁止**写入纳入版本控制的文件、**禁止**经 `/api/config` 等面向公网的响应下发（它与公开的 `CLIENT_SIGN_KEY` 是两个独立值）。worker 对密钥**必须**用常量时间比对。
- **受信真实 IP 头名无内置默认**：`CDN_REAL_IP_HEADER` **必须**可配置且**无内置默认值**；未配置即 fail-safe 到 `CF-Connecting-IP`。实现**禁止**写成 `env.CDN_REAL_IP_HEADER ?? '<默认头名>'`（默认值会让部署即采信某头）。推荐值 `Ali-Cdn-Real-Ip`，经证伪门证实覆写后由运营者显式设置。
- **强制兑现「绝不采信 XFF」**：实现**必须**在解析时拒绝把 `CDN_REAL_IP_HEADER` 取为 `X-Forwarded-For`（不分大小写）——即便运营者误配为该头名，也**必须** fail-safe 到 `CF-Connecting-IP`、**绝不**读取 `X-Forwarded-For`。此举把「绝不采信 XFF」从约定升级为代码强制。
- **部署期证伪门（硬阻断）**：在证伪门证实「客户端发送的密钥头被 CDN 覆写、且受信真实 IP 头被 CDN 覆写」之前，`CDN_REAL_IP_HEADER` **必须**保持未配置。证伪须**证伪式**：携带伪造密钥头（含大小写变体）+ 伪造真实 IP 头 + 伪造 XFF，断言 worker 收到的密钥头等于真实密钥（覆写成立）、解析返回值等于真实出口 IP 且不等于任何伪造值且非多值。
- **运行时校验（纵深防御，非充分）**：采信受信真实 IP 头时**必须**校验其为**单值、合法 IP 字面量（IPv4 或 IPv6）、不含逗号**；多值/含逗号/非法 → 落 `'unknown'`，**禁止**采信。**禁止** `split(',')` 取段再校验（否则透传 `8.8.8.8,<真实>` 首段会通过校验=伪造复现）。
- CDN 侧配置（出站头/回源协议）变更后**必须**重跑部署期证伪门，并与密钥维护同等纪律。
- **回源协议**：阿里云 CDN 回源协议**应**为 HTTPS，使密钥头不在 CDN→源站一跳走明文。

#### 场景：运行时拒绝多值/非法的受信头

- **当** 受信真实 IP 头被采信路径读到，但其值含逗号、为多值、或非合法 IP 字面量
- **那么** 可信真实客户端 IP **必须**为 `'unknown'`
- **并且** **禁止**采信该值（视为覆写假设失效信号）

#### 场景：部署期证伪门证实覆写（非运行时）

- **当** 经 CDN 路径对配置的密钥头/真实 IP 头发送伪造值（含大小写变体）+ 伪造 XFF
- **那么** 部署期证伪门**必须**断言 worker 收到的密钥头 = 真实密钥、且解析返回值 = 真实出口 IP、≠ 任一伪造值、且非多值
- **并且** 仅在该证伪门通过后**才**允许显式配置 `CDN_REAL_IP_HEADER`；未通过则保持未配置（fail-safe 到 CF-IP）
- **并且** 规范**不**声称运行时能区分「单值覆写」与「单值透传」——该保证由本证伪门承担
