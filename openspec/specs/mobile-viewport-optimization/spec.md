## 新增需求

### 需求：Viewport 必须防止输入框聚焦时自动缩放
页面的 viewport meta 标签必须配置为防止浏览器在输入框获得焦点时自动放大页面。必须包含 `maximum-scale=1.0` 和 `viewport-fit=cover` 属性。

#### 场景：iOS Safari 点击输入框
- **当** 用户在 iOS Safari 上点击搜索输入框
- **那么** 页面不会自动放大，保持原始缩放比例

#### 场景：输入完成后缩放恢复
- **当** 用户完成输入并关闭键盘
- **那么** 页面保持 1:1 的缩放比例，不会出现偏移

#### 场景：MathCaptcha 数字输入框
- **当** 用户在 MathCaptcha 弹窗中点击数字输入框
- **那么** 页面不会自动放大

### 需求：所有输入框字体大小必须不小于 16px
页面中所有 input、textarea、select 元素的计算字体大小必须不小于 16px，以防止 iOS Safari 的自动缩放触发机制。

#### 场景：搜索输入框字体大小
- **当** 渲染搜索输入框（SearchBox 组件）
- **那么** 输入框的计算字体大小必须为 16px 或更大

#### 场景：验证码输入框字体大小
- **当** 渲染 MathCaptcha 验证码输入框
- **那么** 输入框的计算字体大小必须为 16px 或更大

### 需求：表单元素必须禁止双击缩放
所有表单交互元素（input、button、select、textarea）必须设置 `touch-action: manipulation`，防止浏览器将快速连续点击解释为缩放手势。

#### 场景：快速连续点击搜索按钮
- **当** 用户在移动端快速连续点击搜索按钮
- **那么** 页面不会触发双击缩放行为

#### 场景：快速连续点击输入框
- **当** 用户在移动端快速连续点击输入框区域
- **那么** 页面不会触发双击缩放行为，输入框正常获得焦点
