## 上下文

`CreateOrderDialog.vue` 中寄件人信息区域有两种状态：
- **已配置**：使用 `.sender-info-compact` 样式，紧凑展示姓名、电话、地址（约 2 行高度）
- **未配置**：使用 `el-empty` 组件（`image-size="40"`），但默认仍有较大的上下间距和描述文字区域，整体高度远大于已配置状态

## 目标 / 非目标

**目标：**
- 缩小 `el-empty` 未配置状态的整体高度，使其与已配置状态视觉协调

**非目标：**
- 不改变 `el-empty` 的功能逻辑（图标、描述文字、「去配置」按钮均保留）

## 决策

**选择**：通过减小 `el-empty` 的 padding 来压缩高度。

**理由**：`el-empty` 的默认 padding 较大，是导致高度不协调的主要原因。通过 scoped CSS 覆盖 padding 即可，无需更换组件。

## 风险 / 权衡

- 无显著风险，仅为样式微调
