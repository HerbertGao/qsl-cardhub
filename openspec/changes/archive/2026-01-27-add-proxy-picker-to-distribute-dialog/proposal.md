# 变更：在分发对话框中添加代领人选择器

## 为什么

当前分发对话框在选择"代领"处理方式时，没有提供代领人选择功能。用户需要手动输入代领人呼号，容易出错且效率低下。通过添加代领人下拉选择器，可以：
- 快速选择本次转卡中的呼号作为代领人
- 支持筛选功能，快速定位目标呼号
- 允许输入不在本次转卡中的呼号（临时代领人），但不保存到后台

## 变更内容

- 在分发对话框中选择"代领"处理方式时，显示代领人下拉选择器
- 下拉选择器的选项为本次转卡（project）的所有呼号（去重）
- 支持筛选功能，用户可以通过输入关键词快速查找呼号
- 支持 `allow-create` 功能，允许用户输入不在本次转卡中的呼号
- 临时输入的呼号仅用于本次分发，不保存到后台数据库

## 影响

- 受影响规范：`card-entry-and-management`（卡片分发功能）
- 受影响代码：
  - `web/src/components/cards/DistributeDialog.vue`：添加代领人选择器组件
  - `src/db/cards.rs`：可能需要添加获取项目下所有呼号的查询函数
  - `src/commands/cards.rs`：可能需要添加获取项目呼号列表的 Tauri 命令
  - `web/src/types/components.ts`：可能需要更新 `DistributeFormData` 和 `DistributeConfirmData` 类型定义
