## 为什么

用户在完成卡片分发工作后，需要将当前转卡项目的卡片数据导出为 Excel 文件，便于存档、统计分析或与他人分享处理结果。目前系统缺少数据导出功能，用户只能通过截图或手动记录的方式保存数据。

## 变更内容

- **新增**：在卡片管理页面添加"导出"按钮
- **新增**：后端实现 Excel (xlsx) 文件生成功能
- **新增**：导出当前项目的卡片列表，包含序号、呼号、数量、状态字段
- **新增**：文件命名规则：`{项目名}_{时间戳}.xlsx`（时间戳格式：YYYYMMDDHHmmss）

## 功能 (Capabilities)

### 新增功能

- `card-export`: 卡片数据导出功能，支持将当前转卡项目的卡片列表导出为 Excel 格式

## 影响

### 代码

- `src/commands/` - 新增导出命令
- `src/db/` - 可能需要新增查询方法（获取导出用数据）
- `ui/src/views/` - 卡片管理页面添加导出按钮

### 依赖

- 需要引入 Rust xlsx 生成库（如 `rust_xlsxwriter` 或 `calamine`）

### API

- 新增 Tauri 命令：`export_cards_to_excel`
  - 输入：`project_id: String`
  - 输出：生成的文件路径或触发文件下载

### 导出字段规格

| 列名 | 数据来源 | 说明 |
|------|----------|------|
| 序号 | `card.serial` | 三位数显示（如 001, 002） |
| 呼号 | `card.callsign` | 原样输出 |
| 数量 | `card.qty` | 支持精确数值或大致显示（≤10, ≤50, >50） |
| 状态 | `card.status` | 导出时的最终状态（待分发/已分发/已退卡） |
