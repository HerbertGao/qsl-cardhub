## 为什么

当前更新检查会从 GitHub Release 获取 `body` 作为更新说明。该内容通常是 Markdown，但前端以纯文本方式渲染，用户会看到 `#`、`-`、代码围栏等源码标记，阅读体验不一致且信息层级不清晰。

## 变更内容

- 将“关于页”中的更新说明展示从纯文本改为 Markdown 渲染。
- 引入安全渲染策略：先清洗再渲染（或等效安全链路），禁止脚本与危险标签注入。
- 仅提供简洁样式（标题、列表、段落、链接、代码块的基础样式），不引入复杂主题。
- 保留现有兜底逻辑：无更新说明时继续展示“无更新说明”。

## 功能 (Capabilities)

### 新增功能

- （无）

### 修改功能

- `auto-updater`: 更新说明的展示行为从“纯文本展示 Markdown 源码”变更为“安全 Markdown 富文本展示”。

## 影响

- 前端页面：`web/src/views/AboutView.vue`（更新说明渲染方式与样式）。
- 前端更新流程：`web/src/services/updateCheck.ts`（字段来源保持不变，仅消费方式改变）。
- 依赖与安全：可能新增 Markdown 渲染/清洗依赖；若离线或受限环境不可安装，则采用内置安全降级方案。
- 测试与验收：需补充典型 Markdown 样例与恶意输入样例的展示验证。
