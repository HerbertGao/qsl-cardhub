## 上下文

web_query_service 使用 Vite 构建前端，配置了 `publicDir: 'static'`。`static` 目录中的文件会在构建时自动复制到输出目录 `public` 的根路径。

## 目标 / 非目标

**目标：**
- 在部署后的站点根目录提供验证文件

**非目标：**
- 不需要动态生成文件内容
- 不需要修改构建配置

## 决策

### 使用 static 目录放置验证文件

将验证文件直接放在 `web_query_service/static/` 目录下。

**理由**：
- Vite 已配置 `publicDir: 'static'`，无需额外配置
- 文件会自动复制到构建输出根目录
- 简单直接，无额外依赖

**替代方案（未采用）**：
- 在构建脚本中动态生成 → 过于复杂，内容是固定的
- 修改 wrangler 配置添加静态资源 → 不必要，现有机制已满足需求

## 风险 / 权衡

无明显风险。文件是静态的，不会影响现有功能。
