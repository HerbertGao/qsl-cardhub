## 上下文

`web_query_service` 查询页当前在结果卡片中仅显示状态徽章与可选备注（`metadata.distribution.remarks`），未展示分发方式与代领人信息。
而同步数据已经包含 `metadata.distribution.method` 与 `metadata.distribution.proxy_callsign`，查询接口也会透传该元数据，因此前端具备直接渲染条件。

约束：
- 该变更应仅调整前端展示，不改接口字段与数据库结构。
- 页面仍需兼容历史数据（distribution 为空或字段缺失）。

## 目标 / 非目标

**目标：**
- 在 `status = distributed` 时，按分发方式展示补充文案：
  - 自取 -> 显示“自取”。
  - 代领 -> 显示“代领（<代领人呼号>）”。
- 保留已有备注展示与复制按钮行为（针对备注）。
- 明确“非已分发状态”不增加分发方式展示。

**非目标：**
- 不新增或修改 `worker` API 响应结构。
- 不改变卡片状态字典与状态流转规则。
- 不在查询页增加新的交互动作（如复制分发方式、跳转代领人）。

## 决策

1. 以 `status === 'distributed'` 作为展示分发方式的主开关。
- 采用原因：仅“已分发”语义与分发结果强绑定，可避免在 `pending/printed/completed` 等状态下出现误导信息。

2. 展示优先级：
- 先显示“分发方式行”（自取 / 代领 + 代领人）；
- 备注存在时继续显示“备注行 + 复制按钮”。
- 采用原因：将“方式”和“备注”拆分，减少把备注当方式的歧义。

3. 代领人缺失时的降级策略：
- `method = 代领` 但无 `proxy_callsign` 时仅显示“代领”。
- 采用原因：兼容历史数据与不完整数据，不阻断展示。

4. 类型定义与模板同步扩展。
- 在 `App.vue` 与 `ResultList.vue` 的 `CardItem` 类型中补齐 `distribution.method/proxy_callsign` 可选字段。
- 采用原因：消除 TypeScript 隐式 any/可选链误用风险。

## 风险 / 权衡

- [风险] 历史数据中 `method` 可能为非标准文本（如“快递”）。
  -> 缓解：仅对“自取/代领”做定制文案；其它值走原值兜底显示或不显示扩展文案。

- [权衡] 非已分发状态不展示分发方式，可能少量场景下用户希望提前看到方式。
  -> 缓解：保持状态语义一致性优先，避免展示“已分发”语境之外的分发结果。
