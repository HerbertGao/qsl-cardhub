# 提案：数量显示增加大致模式

## 为什么

当前系统中所有展示卡片数量的地方（列表、详情、分发、退回、打印标签等）都直接显示精确数值。在某些场景下，不需要暴露精确数量，只需展示大致范围。增加一个"大致"显示模式，将数量归档为 `≤10`、`≤50`、`>50` 三个档位。

## 变更内容

### 1. 全局数量显示模式

新增一个全局设置（localStorage 持久化），控制数量的显示方式：

- **精确**（默认）：保持当前行为，显示精确数值
- **大致**：将数量映射为三个档位：
  - `qty ≤ 10` → 显示 `≤10`
  - `10 < qty ≤ 50` → 显示 `≤50`
  - `qty > 50` → 显示 `>50`

### 2. 影响的显示位置

所有展示 `card.qty` 的前端页面都需要根据模式切换显示：

| 文件 | 说明 |
|------|------|
| `CardList.vue` | 卡片列表页表格"数量"列 |
| `CardDetailDialog.vue` | 卡片详情弹窗 |
| `DistributeDialog.vue` | 分发对话框基本信息区 |
| `ReturnDialog.vue` | 退回对话框基本信息区 |
| `SFOrderListView.vue` | 顺丰订单详情 |
| `CardManagementView.vue` | 录入成功提示文案 |

### 3. 打印标签内容同步

打印 QSL 卡片时，`qty` 字段传入打印接口的值也根据模式变化：

- **精确模式**：标签显示 `QTY: 100`
- **大致模式**：标签显示 `QTY: ≤50`

影响的前端打印调用点：
- `CardManagementView.vue` 中的 `print_qsl` 调用（录入后打印）
- `CardList.vue` 中的 `print_qsl` 调用（列表补打印）

### 4. 模式切换入口

在分发对话框的"数量"显示旁提供切换控件，用户选择后全局生效。

### 5. 数据库与后端

- **数据库不需要变动**：`qty` 字段仍存储精确的 `i32` 值
- **后端不需要变动**：打印接口接收前端传入的字符串，后端只做模板渲染
