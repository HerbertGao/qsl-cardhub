## ADDED Requirements

### 需求：地址智能识别

系统**必须**提供地址智能识别功能，允许用户粘贴一段包含姓名、手机号、省市区、详细地址的自由文本，一键自动拆分并填充到收件人表单各字段。

#### 场景：识别标准格式地址

- **当** 用户在智能识别输入框中粘贴文本 `张三 13812345678 广东省深圳市南山区科技园路1号`
- **那么** 系统**必须**自动识别并填充：姓名=`张三`、手机号=`13812345678`、省份=`广东省`、城市=`深圳市`、区县=`南山区`、详细地址=`科技园路1号`
- **并且** 省市区**必须**通过 AddressSelector 组件正确选中对应选项

#### 场景：识别省略行政后缀的地址

- **当** 用户粘贴文本 `李四 13900001111 广东深圳南山科技园路1号`（省略了"省""市""区"）
- **那么** 系统**必须**仍能正确匹配到广东省、深圳市、南山区
- **并且** 剩余部分 `科技园路1号` 填入详细地址

#### 场景：识别直辖市地址

- **当** 用户粘贴文本 `王五 13700002222 北京市朝阳区建国门外大街1号`
- **那么** 系统**必须**正确识别省份=`北京市`、城市=`北京市`、区县=`朝阳区`、详细地址=`建国门外大街1号`

#### 场景：识别含分隔符的地址

- **当** 用户粘贴的文本以逗号、空格或换行符分隔各部分，例如 `赵六，138-1234-5678，浙江省杭州市西湖区文三路100号`
- **那么** 系统**必须**正确提取手机号（忽略中间横杠）并识别各字段

#### 场景：手机号提取

- **当** 文本中包含符合 `1[3-9]` 开头的 11 位数字序列（可能被横杠或空格分隔）
- **那么** 系统**必须**提取该序列作为手机号
- **并且** 提取的手机号**必须**去除横杠和空格后为纯 11 位数字

#### 场景：识别失败时的处理

- **当** 系统无法从文本中识别出省市区信息
- **那么** 系统**必须**将手机号以外的文本中较短部分（≤5字符）作为姓名候选、其余作为详细地址候选填入表单
- **并且** 省市区字段保持为空，用户可手动选择
- **并且** 系统**禁止**弹出错误提示，仅进行最大努力填充

#### 场景：输入为空时点击识别

- **当** 用户在智能识别输入框为空时点击「智能识别」按钮
- **那么** 系统**禁止**执行任何填充操作
- **并且** 系统**必须**提示用户先输入地址文本
