# 前端界面操作优化提案

## 概述

本提案旨在优化 QSL-CardHub 的前端界面操作体验，通过合并相关概念和功能、重新组织界面布局和菜单结构，提升用户的工作效率。

## 为什么

### 当前问题

1. **概念冗余**：「打印配置」中的「任务名称」与「卡片管理」中的「转卡项目」在业务上是同一概念，但分布在两个不同的地方，造成用户认知负担。

2. **功能分散**：打印标签功能单独在「打印」页面，录入卡片功能在「卡片管理」页面，用户需要频繁切换页面才能完成"录入→打印"的完整工作流。

3. **操作布局不合理**：
   - 查看按钮占据独立列，但使用频率不如分发功能
   - 呼号是关键信息，但缺少快速复制功能
   - 打印面单功能被埋在「更多」菜单深处

4. **配置过于复杂**：支持多配置但实际使用中用户通常只需要一套配置，增加了不必要的复杂性。

5. **菜单结构混乱**：「打印」「打印配置」「打印模板」三个独立菜单，层级不清晰。

## 变更内容

### 变更 1：合并「任务名称」与「转卡项目」概念

- 移除打印配置中的「任务名称」字段
- `task_name` 统一重命名为 `project_name`
- 打印时从卡片所属项目获取 `project_name`

### 变更 2：合并打印功能到卡片录入

- `CardInputDialog.vue`：增加打印标签复选框、序列号功能（包括跳过4）
- 删除 `PrintView.vue` 页面
- 移除「打印」菜单入口

### 变更 3：优化卡片列表操作布局

- 卡片表添加序列号字段（`serial` INTEGER）
- 呼号列后添加查看按钮和复制呼号按钮
- 分发按钮移到显眼位置
- 操作菜单新增「打印标签」功能

### 变更 4：重构菜单结构和配置管理

**菜单结构变更**：
- 移除独立「打印」菜单项
- 「打印配置」改为二级菜单，包含「打印机配置」和「标签模板配置」
- 移除独立「打印模板」菜单项

**配置管理变更**：
- 从多配置（Profile 列表）改为单配置
- 移除配置的 CRUD 操作，改为直接编辑单一配置

### 变更 5：通用序列号格式化

- 新增 `formatSerial()` 通用工具函数
- 统一所有序列号显示为3位前导零格式
