## 1. 会话/PoW 纯函数（src/worker/session.js，便于单测）

- [x] 1.1 PoW 校验：`powLeadingZeroBits(hashHex)` + `verifyPow(seed, nonce, difficulty)`（`sha256(seed+":"+nonce)` 前导零 bit ≥ difficulty）。边界：difficulty 0/超高、nonce 空/非串。
- [x] 1.2 会话 token 构造/校验：`makeToken(sid, SESSION_SECRET)`=`sid.HMAC` / `parseToken(token, SESSION_SECRET)`→sid 或 null。HMAC **必须**用真 HMAC-SHA256（`crypto.subtle`，禁 `sha256(secret+msg)`）；`sid` 字符集限 hex/base64url（不含 `.`）。`parseToken` **必须**按顺序：① `lastIndexOf('.')` 切分 sid 与 HMAC（无 `.`→null）→ ② 校验 sid 字符集 ∈ hex/base64url（不过即返 null，先于 HMAC）→ ③ 再验 HMAC（不过返 null）；加单测覆盖（无 `.`、sid 含非法字符在 HMAC 验证前即 null、HMAC 不符→null）。
- [x] 1.3 会话签名校验（会话 sk）：`_sig = HMAC-SHA256(sk, canonicalPayload)`（禁 `sha256(payload+sk)`）；`canonicalPayload` **必须**按固定顺序用 `\n` 连接五项：`sid` + 路径 + 实际携带**业务**查询参数按 key 字母序的已 URL 编码串 `key=value&...`（**排除 `_sig`、`_ts`、`_nonce`**——`_ts`/`_nonce` 由后两项专门承载、不在此串重复）+ `_ts` + `_nonce`（即签 `_ts`/`_nonce`、纳入 `sid` 非完整 token，绑定新鲜度/防重放进 HMAC、禁换 ts/nonce 复用旧 `_sig`）；路径式端点字段③为空串（`_ts`/`_nonce` 仍由后两项承载）；抽 `verifySessionSig(url, sk, ...)`（_ts 时窗 + 调用方传入 nonce 防重放结果）；`canonicalPayload` **必须**为 worker 与 client 共用的**单一事实源模块**（**禁止**两处各写易漂移实现）；落法**必须**保证 `pnpm run build`（`vue-tsc --noEmit`）仍绿——采用**共享 `.js` 模块 + 同名手写 sibling `.d.ts`**（worker `import` 该 `.js`、client `utils/sign.ts` re-export 该 `.js`，`.d.ts` 提供类型，**无需**改 tsconfig），或等价地启用 `allowJs` 并将该模块路径纳入 `tsconfig` `include`（二选一，apply 时择一）。
- [x] 1.4 自适应难度：`difficultyFor(rate)`=clamp(BASE + tier(rate), BASE_MIN, DIFF_MAX)——加 `BASE_MIN>0` 下限（恒有真实 PoW）+ `DIFF_MAX` 封顶（手机可解）；tier 单调递增（每超阈值 +2~4 bit）受封顶。频率计数键经 `clientBindingKey`（IPv6 /64、IPv4 /32）归桶。`powrate` 读失败/超时→`difficulty` 取 `DIFF_MAX`（fail-secure，禁跌回 `BASE_MIN`）。注释标注手机耗时区间。
- [x] 1.5 UA 指纹 `uaHash(ua)`=sha256 截断；会话绑定校验 `sessionValid(session, ip, ua, now)`（exp/ua_hash 必判；IP 判按 `session.binding_mode` 分支：`ip` 模式校 `clientBindingKey 归一键 ip === session.ip 归一键`，`none` 模式即 unknown 会话**跳过 IP 比对**）。
- [x] 1.6 IP 归一键 `clientBindingKey(ip)`（新增于 `src/worker/client-ip.js`，纯函数）：IPv4 直通（/32 原值）；IPv6 取前 64 位归一（/64 前缀）；`unknown` 直通（返回 `unknown`）。+ 单测（IPv4/32、IPv6/64、压缩形式、unknown）。供 index.js 与 session.js import 作为全套按-IP 键（seed.challenge_ip、session.ip、powrate、ratelimit:session、查询/兑换 IP 比对）的唯一归一口径。
- [x] 1.7 建立共享 canonical 模块（单一事实源、禁两份漂移）：以**共享 `.js` 模块 + 同名手写 sibling `.d.ts`**，或**启用 `allowJs` 并将该模块路径纳入 `tsconfig` `include`**（二选一，apply 时择一）落地 `canonicalPayload`，worker `import` 该模块、client `utils/sign.ts` re-export 该模块；确保 `vue-tsc --noEmit`（`pnpm run build`）通过（client tsconfig `strict:true`、默认无 `allowJs`、glob 仅 `.ts/.tsx/.vue`，TS re-export 裸 `.js` 需 `.d.ts` 提供类型避免 TS7016，或经 `allowJs`+`include` 纳入）。

## 2. Worker 端点与接线（src/worker/index.js）

- [x] 2.1 `GET /api/session/challenge`：令 `bkey = clientBindingKey(getClientIP)` → 前置独立桶限流 `ratelimit:session:<bkey>`（独立于查询桶）→ 生成 seed（高熵）+ `difficultyFor(powrate:<bkey>)` → KV `powseed:<seed>`=`{difficulty, challenge_ip:bkey, challenge_ua_hash, exp}`(TTL~2min) + `powrate:<bkey>` 递增 → 返回 `{seed, difficulty}`。**KV 读写各处包 try/catch、区分 KV 未绑 vs 运行时失败，禁冒泡到顶层 catch 变 500**：KV 未绑→503；`powseed` 写入运行时失败/超时→fail-closed 503（禁返回不可兑换的 seed）；`powrate` 读写失败/超时→`difficulty` 取 `DIFF_MAX`（fail-secure，禁跌回 `BASE_MIN`）；`ratelimit:session` 读写失败/超时→握手端点 fail-closed 503——**fail 方向由调用方传入，禁复用查询桶 `checkRateLimit` 的 fail-open 默认值 / 禁原语内硬编码单一 fail 方向**（否则诱发握手桶读失败即继承 fail-open 绕开握手限流）。
- [x] 2.2 `POST /api/session`：令 `bkey = clientBindingKey(getClientIP)` → 前置独立桶限流 `ratelimit:session:<bkey>` → 读 KV `powseed:<seed>`（无→拒）→ **立即删** → `verifyPow` → 校验当前 `bkey`===`challenge_ip`(归一键) 且 sha256(当前 UA)===`challenge_ua_hash`（不等→拒）→ 生成 sid+sk → KV `session:<sid>`={ua_hash, exp, sk, ...绑定}(TTL~10min)：**非 unknown**：`binding_mode=ip`、`session.ip=clientBindingKey(getClientIP)`（=bkey）；**unknown**（bkey=='unknown'）：`binding_mode=none`、`session.ip=null`（sessionValid 跳过 IP 比对） → 返回 `{token, sk, exp, quota}`（`unknown` 来源 quota 取压低的 `QUOTA_unknown`，见 2.7）。**每处 KV 读写包 try/catch、区分 KV 未绑 vs 运行时失败，禁冒泡到顶层 catch 变 500**：反滥用关键键（seed/session）KV 未绑或运行时读写失败/超时→fail-closed 503；`ratelimit:session` 读写失败/超时→握手端点 fail-closed 503——**fail 方向由调用方传入，禁复用查询桶 `checkRateLimit` 的 fail-open 默认值 / 禁原语内硬编码单一 fail 方向**。
- [x] 2.3 改查询端点（约 557-576）：删 `verifySignature(CLIENT_SIGN_KEY)`；**Layer0 纯 IP 限流 `checkRateLimit(ratelimit:<bkey>)` 置于会话校验之前**（`bkey = clientBindingKey(getClientIP)`；保持 fail-open）——使「无会话但限流通过」的请求落到会话校验返回确定的 **401**、限流触发返 **429**（作为 5.2a 哨兵：未限流=401 / 已限流=429）；再「HMAC 有效 ∧ KV 命中 session ∧ 绑定项全过」三要素与（parseToken→KV session→sessionValid(ip=bkey, ua)：按 `session.binding_mode` 分支——`ip` 模式以归一键比对 `bkey===session.ip`、`none` 模式（unknown 会话）跳过 IP 比对仅校 exp/ua_hash），禁 token-only 快路径；→verifySessionSig(sk)+nonce 防重放→KV `sessionq:<sid>` 配额（超→429）。无/失效会话→401。**每处 KV 读写包 try/catch、区分 KV 未绑 vs 运行时失败，禁冒泡到顶层 catch 变 500**：nonce 防重放/配额/session 的 KV 运行时读写失败/超时→fail-closed（401/503）；唯查询桶 `ratelimit:<bkey>`（checkRateLimit）保持 fail-open。
- [x] 2.7 `clientBindingKey(getClientIP)=='unknown'` 分支：会话签发对 unknown 来源取最高难度档、**不**绑 IP（仅 UA+配额），unknown 用独立计数桶（key=`unknown`，即 clientBindingKey 对 unknown 直通），禁污染具体 IP 桶；**该会话配额取压低的 `QUOTA_unknown`（如 ≤ 3，远小于常规 `QUOTA`≈50）**——unknown 不绑 IP 可跨 IP 搬移，压低配额使一次最高难度 PoW 仅换极少次查询、搬移价值趋零。
