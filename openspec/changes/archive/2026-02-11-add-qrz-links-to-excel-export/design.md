## 上下文

当前 Excel 导出由 `src/commands/export.rs` 的 `generate_excel` 统一生成，列结构固定为 4 列。
项目已经在卡片元数据中维护地址缓存：`metadata.address_cache`，并按来源区分 `qrz.cn` 与 `qrz.com`。

本次变更要求：导出新增三列时直接读取缓存，不触发在线查询；若没有对应缓存则留空。

## 目标 / 非目标

**目标：**
- 新增三列：`QRZ.cn(中文)`、`QRZ.cn(English)`、`QRZ.com`。
- 导出仅从卡片缓存读取值，不依赖网络。
- 缺失缓存时输出空单元格，保持导出流程稳定。

**非目标：**
- 不在导出阶段调用 QRZ.cn / QRZ.com 接口。
- 不调整缓存写入逻辑与缓存有效期策略。
- 不变更导出命令参数或前端交互。

## 决策

1. 读取来源映射固定化。
- `qrz.cn.chinese_address` -> `QRZ.cn(中文)`
- `qrz.cn.english_address` -> `QRZ.cn(English)`
- `qrz.com.english_address` -> `QRZ.com`

2. 缺失即空，不兜底拼 URL。
- 方案 A（采用）：缓存缺失写空字符串。
- 方案 B（不采用）：缓存缺失时拼接默认查询 URL。该方案与“仅使用缓存”的要求冲突。

3. 在导出模块内增加缓存提取辅助逻辑。
- 对单张卡片统一提取三列值，降低主循环复杂度。

4. 用单元测试覆盖缓存命中/未命中场景。
- 确保三列顺序稳定。
- 确保无缓存时三列为空。

## 风险 / 权衡

- [风险] 历史数据可能只有部分来源缓存，导致三列局部为空。
  -> 缓解：按列独立填充，允许部分为空，不影响导出成功。
- [权衡] 不再导出可点击查询链接。
  -> 换取与缓存数据一致、可离线导出的行为。
