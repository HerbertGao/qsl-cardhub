## 新增需求

### 需求：分发对话框打印地址功能
分发对话框必须提供打印地址按钮，允许用户直接打印地址信息。

#### 场景：显示打印地址按钮
- **当** 用户在分发对话框中查看地址缓存
- **那么** 每个地址项的复制地址按钮后必须显示打印地址按钮
- **并且** 按钮图标为打印机图标
- **并且** 按钮文本为"打印地址"

#### 场景：打印地址按钮点击
- **当** 用户点击打印地址按钮
- **那么** 系统必须获取当前地址项的所有地址信息
- **并且** 根据地址源类型格式化地址数据：
  - **qrz.cn**：使用 `chinese_address` 和 `english_address`
  - **qrz.com**：使用 `english_address`（可能包含姓名）
  - **QRZ卡片查询**：使用 `name`、`english_address` 和 `mail_method`
- **并且** 调用 `print_address` 命令进行打印
- **并且** 显示打印成功或失败的提示消息

#### 场景：地址数据格式化
- **当** 系统准备打印地址时
- **那么** 必须根据地址源类型格式化地址数据：
  - 如果存在姓名，必须包含姓名行
  - 如果存在中文地址，优先使用中文地址
  - 如果只有英文地址，使用英文地址
  - 如果存在邮寄方式，必须包含邮寄方式信息
  - 必须包含呼号信息（用于标识）
- **并且** 必须不包含日期信息

#### 场景：打印地址成功反馈
- **当** 地址打印成功时
- **那么** 系统必须显示成功提示消息（如"地址打印成功"）
- **并且** 提示消息必须包含呼号信息

#### 场景：打印地址失败处理
- **当** 地址打印失败时
- **那么** 系统必须显示错误提示消息
- **并且** 错误消息必须包含失败原因
- **并且** 允许用户重试打印

### 需求：地址打印命令
系统必须提供地址打印命令，支持使用地址模板打印地址信息。

#### 场景：调用地址打印命令
- **当** 前端调用 `print_address` 命令
- **那么** 系统必须接收以下参数：
  - `printer_name`：打印机名称
  - `address_data`：地址数据对象（包含姓名、中文地址、英文地址、邮寄方式、呼号等）
- **并且** 必须加载 `address.toml` 模板
- **并且** 必须使用地址数据填充模板元素
- **并且** 必须生成双份打印布局（如果模板配置了 `duplicate_print = true`）
- **并且** 必须调用打印流程发送到打印机

#### 场景：地址数据验证
- **当** 系统接收地址打印请求时
- **那么** 必须验证地址数据中至少包含以下之一：
  - 中文地址
  - 英文地址
- **并且** 必须验证呼号字段存在
- **并且** 如果验证失败，返回描述性错误信息

#### 场景：地址模板加载
- **当** 系统执行地址打印时
- **那么** 必须从 `config/templates/address.toml` 加载模板配置
- **并且** 如果模板文件不存在，返回错误信息
- **并且** 如果模板格式错误，返回描述性错误信息
