## 新增需求

### 需求：导出范围必须覆盖项目全部卡片

导出到 Excel 时，系统必须导出当前项目（在当前筛选条件下）的**全部**卡片，导出结果的数据行数必须等于实际卡片数，禁止受任何分页页大小上限的限制而被截断。

#### 场景：卡片数超过分页上限时仍全量导出

- **当** 当前项目录入了 415 张卡片
- **并且** 用户点击「导出」并完成文件保存
- **那么** 导出的 Excel 工作表必须包含 416 行（1 行表头 + 415 行数据）
- **并且** 数据行数必须等于项目实际卡片数，不得停留在 100

#### 场景：导出行数随实际卡片数变化

- **当** 当前项目有 N 张卡片（N 可大于分页页大小上限）
- **并且** 用户导出该项目
- **那么** 导出文件的数据行数必须恰好为 N

#### 场景：导出不依赖分页页大小上限

- **当** 系统执行导出查询
- **那么** 必须使用不施加 `page_size` 上限的全量查询路径获取卡片
- **并且** 卡片列表页（UI）的分页与页大小上限行为必须保持不变、不受本需求影响
